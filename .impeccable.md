## Design Context

### Users

Researchers, educators, and critical listeners use trackswitch.js to present and compare scientific audio results. They need fast, exact playback navigation while concentrating on musical and analytical differences.

### Brand Personality

Precise, restrained, and dependable. The interface should feel like a focused scientific listening instrument rather than an entertainment-first music player.

### Aesthetic Direction

Preserve the established compact light visualization surfaces, dark control bar, orange accent, and utilitarian typography. New controls should be quiet at rest, legible when engaged, and consistent across default and sync modes.

### Design Principles

- Keep playback and comparison as the strongest visual hierarchy.
- Reveal annotation detail through interaction instead of adding resting clutter.
- Use existing colors, spacing, typography, and control geometry.
- Make precise actions equally understandable with pointer, touch, and keyboard input.
- Report invalid scientific data explicitly; never silently alter or discard it.
